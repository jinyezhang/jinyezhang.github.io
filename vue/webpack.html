# 1.目录结构

  1. 在文件夹中创建 3 个文件夹，分别是：src dist config
  2. 在src文件夹中创建一个main.js
  3. 在dist文件夹中创建一个index.html
  4. 在config文件夹中创建一个webpack.dev.js
  5. 在根目录文件夹中，输入npm init -y,系统会生成一个package.json,并且有相关的一些配置内容
  6. 使用命令行，输入git init，系统会生成一个git文件夹，该文件是隐藏不可见的

# 2.全局安装 webpack webpack-cli webpack-dev-server

npm install webpack webpack-cli webpack-dev-server -g

注意：

  1. -g 就是全局安装
  2. 安装完一定要注意版本号，我在安装的时候就出现了各种报错问题，根本原因就是版本不兼容

# 3.查看webpack的版本——处理兼容问题

webpack -v

输入webpack -v可以查看到webpack webpack-cli webpack-dev-server的版本号 注意：webpack-cli需要使用3.3.12版本的，所以需要重新安装低版本的webpack-cli 安装完成后，对应的package.json文件中也会有相应的改变：

# 4.配置webpack.dev.js文件

配置config文件夹中的webpack.dev.js文件，定义入口文件为main.js，打包环境为开发环境development，出口文件为dist文件夹，如下示例：

const path = require("path"); //这里引入node的path模块
module.exports = {
  //入口文件可以有多个，多个的时候用数组的形式写
  entry: {
    main: "./src/main.js", //这里定义main.js为入口文件
  },
  //打包环境，开发/生产
  mode: "development", //打包环境为：生产环境
  //出口，只能由一个
  output: {
    filename: "[name]-bundle.js", //打包后的文件名称，[name]是根据打包的文件名自己生成的，后面的-bundle是写死的
    path: path.resolve(__dirname, "../dist"), //dist文件夹
    publicPath: "/", //公开路径
  },
  //配置本地服务器
  devServer: {
    contentBass: "dist", //指定一个本地服务器路径，否则会默认为根目录
  },
};

此时并没有在dist文件夹中生成main.js文件，而且控制台报错如下： 百度后发现解决方法如下：

webpack ./src/main.js -o ./dist

此时发现dist文件夹中出现了一个main.js文件，然后在index.html文件中引入main.js文件 在index.html文件中，右击选择open with live server即可打开此文件 此时，在main.js文件中添加内容，然后保存，发现打开的页面控制台中有打印相应的内容。
