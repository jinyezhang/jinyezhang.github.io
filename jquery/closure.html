# 闭包定义

提示

闭包就是能够读取其他函数内部变量的函数。例如在 javascript 中，只有函数内部的子函数才能读取局部变量，所以闭包可以理解成“定义在一个函数内部的函数。在本质上，闭包是将函数内部和函数外部连接起来的桥梁。

# 我遇到的闭包场景

for循环中，如果给每个元素都使用函数进行处理的话，很可能出现闭包问题

# 简单的 for 循环代码

var arr = [0, 1, 2, 3, 4, 5];
function fn() {
  for (var i = 0; i < arr.length - 1; i++) {
    console.log("索引", i);
  }
}

# for 循环中加入 setTimeout 函数

var arr = [0, 1, 2, 3, 4, 5];
function fn2() {
  for (var i = 0; i < arr.length; i++) {
    setTimeout(() => {
      console.log("索引", i);
    }, 100);
  }
}

控制台打印效果如下： 发现所有的索引打印出来都是 6. 这是因为setTimeout是一个异步函数，在for循环执行完循环后，此时的i是 5 的情况下，arr.length是 6.则再次进行++，然后，异步的setTimeout延时函数可以执行了，此时的i就是 6 了，所以会打印 6 次索引6。

如果仅仅是为了保证索引能够依次执行到函数中的话，则可以有两种方式处理：

# 1.for 循环中用 let 代替 var——解决

代码如下：

var arr = [0, 1, 2, 3, 4, 5];
function fn3() {
  for (let i = 0; i < arr.length; i++) {
    setTimeout(() => {
      console.log("索引", i);
    }, 100);
  }
}

# 2.for 循环中使用立即执行函数包裹——解决

代码如下：

var arr = [0, 1, 2, 3, 4, 5];
function fn4() {
  for (let i = 0; i < arr.length; i++) {
    ((i) => {
      setTimeout(() => {
        console.log("索引", i);
      }, 100);
    })(i);
  }
}

# for 循环中加入 click 点击事件

如果跟上面一样，都只是为了一个索引的话，或者索引对应的其他参数的话，其实可以直接通过将var改为let来解决。

但是如果还有其他的操作，这种方式就不行了。只能是通过立即执行函数的方式来解决了
